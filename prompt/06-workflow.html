实战工作流：迭代、评审、复用与提示词库

06. 实战工作流：迭代、评审、复用与提示词库

提示词的终点不是“这次能出结果”，而是：下次还能稳定复现。

本章提供一套把提示词当作“资产”来管理的方法：版本化、评审、A/B 对比、模板化、以及建立自己的 Prompt Library。

1. 最小可行工作流（MVP）

每次写提示词，建议按 4 步走：

  1. 定目标：这次生成的“可交付物”是什么？（文章/分镜表/海报图）

  2. 定结构：输出必须是可检查的结构（标题/表格/JSON）

  3. 跑一版：不要一口气加很多约束

  4. 做对比：只改一个变量做 A/B 测试

你要追求的是：

  • 可检查：有没有缺字段？有没有违约束？

  • 可复现：下次换素材还能用。

2. 迭代三板斧：加、减、锁

  • 加（Add）：当输出跑偏，就补充约束与例子。

  • 减（Remove）：当输出僵硬或发散，就删掉无效/冲突的限制。

  • 锁（Lock）：当某个写法稳定，把它固化成“模板段落”。

常见可锁定段落：

  • 受众约束（小学生/专业人士）

  • 输出结构（表格/JSON）

  • 自检清单（质量门槛）

3. 质量评审：用“检查清单”替代主观感觉

3.1 文字输出评审清单

  • [ ] 结构完整（标题/小结/行动清单）

  • [ ] 受众匹配（术语密度、例子难度）

  • [ ] 无明显事实错误（若不能确定，明确“不确定”）

  • [ ] 关键点覆盖（必须点全出现）

  • [ ] 语言清晰（少空话、少套话）

3.2 图片输出评审清单

  • [ ] 主体明确（没有跑题）

  • [ ] 风格一致（不混风）

  • [ ] 构图达标（主体位置、景别）

  • [ ] 手/脸质量（常见崩坏点）

  • [ ] 无水印/文字（按需求）

3.3 视频输出评审清单

  • [ ] 主角一致（外观/服装/标志物）

  • [ ] 场景连续（没有突兀跳变）

  • [ ] 动作可连（不会瞬移）

  • [ ] 节奏符合时长（每个 beat 可理解）

  • [ ] 镜头语言清晰（推拉摇移有依据）

4. 提示词版本化：像写代码一样写 Prompt

建议每个可复用提示词都带一个“头部元数据”，例如：

# Prompt: KidsExplainConcept v1.2
# Use-case: 面向小学四年级科普解释
# Inputs: {concept}
# Outputs: explanation + summary + quiz
# Notes: 禁术语；必须类比；最后给练习题

然后正文用模板变量 {} 标记。

5. Prompt Library（提示词库）怎么建

最简单的提示词库目录结构：

  • prompts/text/：写作、总结、改写、教学

  • prompts/novel/：悬疑、科幻、奇幻、言情、武侠

  • prompts/image/：写实、动漫、水彩、赛博、像素

  • prompts/video/：口播、剧情、广告、分镜

每个 prompt 文件至少包含：

  • 用途（Use-case）

  • 变量说明（Inputs）

  • 输出结构（Outputs）

  • 1 个最小示例（Example）

6. 一条非常实用的“二段式用法”

当你要做复杂任务（比如视频分镜 + 文案 + 出图 prompt），不要一次让模型全做完。

推荐两段式：

  1. 先让模型产出结构化计划（大纲/镜头表/字段齐全）

  2. 再让模型按计划逐项生成（每次只生成一部分）

这样可控性更强，且方便人工审稿。
