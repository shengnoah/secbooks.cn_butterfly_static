OpenClaw 经典 Skills 清单与用法（默认布局）

发表于2026-02-20|更新于2026-02-26|AIGCOpenClaw
|阅读量:

本文不使用 book layout，采用默认文章布局，整理 OpenClaw 里最常用的一批“经典 skills”，并给出典型用法与组合套路。

  • OpenClaw Repo：https://github.com/openclaw/openclaw
  • 官方文档（Skills）：https://docs.openclaw.ai/tools/skills

注：skills 列表会随版本变化；本文以 openclaw/openclaw 仓库 skills/*/SKILL.md 为参考。

0. Skills 的通用用法（先看这一段就够）

0.1 Skills 从哪里加载

常见加载优先级（同名覆盖）通常是：

  1. <workspace>/skills（项目/工作区私有）

  2. ~/.openclaw/skills（managed skills，共享）

  3. bundled（随安装自带）

0.2 常用命令

openclaw skills list
openclaw skills list --eligible
openclaw skills info <name>
openclaw skills check

0.3 选择“经典 skills”的标准

  • 能覆盖高频工作流：写代码、查仓库/Issue、做总结、记笔记、可观测/审计、消息通知

  • 能与其他 skill 组合形成闭环

1. coding-agent：把“写代码”变成可并行的背景任务

  • Skill 文档：https://raw.githubusercontent.com/openclaw/openclaw/main/skills/coding-agent/SKILL.md

  • 适用：重构/生成测试/批量改动/新功能从 0 到 1。

典型套路：

  • 主 agent 负责拆任务与验收标准

  • coding-agent 在独立目录（建议临时 git repo / worktree）里执行，避免污染主工作区

示例（思路级）：

  • 让 agent 在临时目录生成实现，再把 diff 合回主分支。

2. github：用 gh CLI 做 PR/Issue/CI 的“可脚本化运营”

  • Skill 文档：https://raw.githubusercontent.com/openclaw/openclaw/main/skills/github/SKILL.md

  • 依赖：本机已安装并登录 gh。

典型场景：

  • 列 PR、看 CI、拉取 run logs

  • 创建/评论 Issue

建议：

  • 配合 summarize：把 PR/Issue 的讨论串做摘要归档

  • 配合 session-logs：把关键操作记录下来

3. gh-issues：专注 Issue 的收集、筛选与跟进

  • Skill 文档：https://raw.githubusercontent.com/openclaw/openclaw/main/skills/gh-issues/SKILL.md

典型场景：

  • 每天自动拉取“需要我处理”的 issues

  • 自动生成“今日待办（triage）”

4. obsidian：把 OpenClaw 的产出沉淀到知识库

  • Skill 文档：https://raw.githubusercontent.com/openclaw/openclaw/main/skills/obsidian/SKILL.md

典型场景：

  • 把多渠道消息/执行记录写入 Obsidian Daily Note

  • 产出项目日志：会议纪要/决策记录/变更说明

组合建议：

  • summarize → 生成摘要

  • obsidian → 写入知识库（形成长期可检索资料）

5. summarize：通用“信息压缩器”（会议/聊天/网页/日志都能用）

  • Skill 文档：https://raw.githubusercontent.com/openclaw/openclaw/main/skills/summarize/SKILL.md

典型场景：

  • 总结一段长聊天记录

  • 总结工具输出（日志/命令行）

  • 生成要点/行动项（Action Items）

建议输出格式（可复用模板）：

  • TL;DR

  • 关键结论

  • 风险点

  • 下一步（Owner/DDL）

6. session-logs：把每次会话/执行过程留痕，方便复盘与审计

  • Skill 文档：https://raw.githubusercontent.com/openclaw/openclaw/main/skills/session-logs/SKILL.md

典型场景：

  • 记录每次 OpenClaw 会话的关键输入输出

  • 对外汇报/复盘时可快速回放

建议：

  • 配合 model-usage 做“成本 + 产出”的对比

7. model-usage：用量/成本观测（把 AI 当工程资源管理）

  • Skill 文档：https://raw.githubusercontent.com/openclaw/openclaw/main/skills/model-usage/SKILL.md

典型场景：

  • 查看近期模型使用情况

  • 找出异常增长（提示词膨胀、工具调用循环等）

8. tmux：管理交互式会话（适合跑长任务/多窗口协作）

组合建议：

  • tmux 捕获输出 → summarize 压缩 → obsidian 归档

9. slack / discord：把 OpenClaw 接入团队沟通渠道

  • Slack skill 文档：https://raw.githubusercontent.com/openclaw/openclaw/main/skills/slack/SKILL.md

  • Discord skill 文档：https://raw.githubusercontent.com/openclaw/openclaw/main/skills/discord/SKILL.md

典型场景：

  • 定时发送日报/告警/状态更新

  • 在频道中把 issue/PR 的变更自动播报

建议：

  • 先做“只读/只通知”能力（降低误操作风险）

  • 再逐步开放“写/改/删/管理”能力

10. notion：结构化笔记/任务库（适合团队协作）

  • Skill 文档：https://raw.githubusercontent.com/openclaw/openclaw/main/skills/notion/SKILL.md

典型场景：

  • 把会议纪要写入 Notion 数据库

  • 自动更新项目状态（Roadmap/OKR）

11. weather：最简单但很实用的“外部信息工具”

  • Skill 文档：https://raw.githubusercontent.com/openclaw/openclaw/main/skills/weather/SKILL.md

典型场景：

  • 出差/通勤前查询天气

  • 结合日程做提醒（可与消息渠道组合）

12. 推荐组合拳（抄作业版）

12.1 工程研发闭环

  1. github / gh-issues：收集需求与上下文

  2. coding-agent：并行产出实现

  3. session-logs：留痕

  4. summarize：输出 PR 摘要 + 风险点

  5. slack/discord：通知团队

12.2 个人知识管理闭环

  1. tmux 抓输出/过程

  2. summarize 压缩成“可复用知识”

  3. obsidian / notion 入库
