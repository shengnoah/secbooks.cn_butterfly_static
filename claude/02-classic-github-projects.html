Claude 教程：经典 GitHub 项目清单（含用途与上手）

本文按「官方 SDK / Claude Code 生态 / MCP 生态 / 可视化 / 网关与观测」分类整理一些常用、口碑较好的 Claude 相关项目，并给出最小上手方式。

说明：星标数量会随时间变化；以“是否常用/是否代表性/是否能解决实际问题”为主。

1. 官方 SDK / Cookbook（推荐优先看）

1.1 anthropics/claude-cookbooks（Cookbook）

  • 仓库：https://github.com/anthropics/claude-cookbooks

  • 用途：官方示例/Notebook 合集，覆盖提示词模式、工具调用、RAG、结构化输出等常见需求。

  • 适合你：需要“可运行的配方”来验证某个能力或快速抄作业。

上手建议：

  1. 先按 README 把示例跑通。

  2. 把你项目里的“业务输入/输出”替换进去，形成自己的最小模板。

1.2 anthropics/anthropic-sdk-python（Python SDK）

  • 仓库：https://github.com/anthropics/anthropic-sdk-python

  • 用途：在 Python 服务/脚本里调用 Claude API。

最小上手（示意）：

pip install anthropic
from anthropic import Anthropic

client = Anthropic()  # 通过环境变量配置密钥
msg = client.messages.create(
    model="claude-3-5-sonnet-latest",
    max_tokens=256,
    messages=[{"role": "user", "content": "用一句话解释 MCP 是什么"}],
)
print(msg.content)

1.3 anthropics/anthropic-sdk-typescript（TypeScript SDK）

  • 仓库：https://github.com/anthropics/anthropic-sdk-typescript

  • 用途：Node.js/TS 项目里调用 Claude API（适合 Web/Serverless/工具链）。

最小上手（示意）：

npm i @anthropic-ai/sdk
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY! });

const msg = await client.messages.create({
  model: "claude-3-5-sonnet-latest",
  max_tokens: 256,
  messages: [{ role: "user", content: "给我一个 10 行的 TypeScript 示例" }],
});

console.log(msg.content);

2. Claude Code 生态（插件 / 模板 / 用量分析）

这一类通常解决：多任务/多会话管理、审计与回放、用量统计、工作流模板化。

2.1 thedotmack/claude-mem（Claude Code “记忆”插件）

  • 仓库：https://github.com/thedotmack/claude-mem

  • 用途：自动捕获 Claude Code 会话内容，并用模型做“压缩记忆/摘要”，让后续会话更可持续。

  • 适合你：长项目、多天迭代、上下文容易爆、想把“做过什么”系统化沉淀。

上手建议：

  • 按项目 README 安装为 Claude Code 插件；

  • 配合你团队的约定（例如：每次合并前/每天收工时生成一次摘要）。

2.2 ryoppippi/ccusage（Claude Code 用量统计）

  • 仓库：https://github.com/ryoppippi/ccusage

  • 用途：解析 Claude Code 本地 JSONL 日志，统计 token/成本/模型用量等。

  • 适合你：想把 Claude Code 当“工程资源”做成本管理/用量看板。

典型用法：

  • 统计近 7 天用量，找出异常飙升的会话；

  • 对比不同提示词模板的成本差异。

2.3 davila7/claude-code-templates（模板与工作流）

  • 仓库：https://github.com/davila7/claude-code-templates

  • 用途：提供 Claude Code 的模板化能力（把常用任务“固化成模板/命令”）。

  • 适合你：重复任务多（代码审计、重构、生成测试、写文档），想要“一键化”。

建议：

  • 先做 3 个模板：

    • “从需求生成实施计划”；
    • “变更后补测试 + 解释边界”；
    • “PR 自检清单”。

2.4 ComposioHQ/awesome-claude-plugins（插件生态清单）

  • 仓库：https://github.com/ComposioHQ/awesome-claude-plugins

  • 用途：收录 Claude Code 插件/自定义命令/hooks/MCP servers。

  • 适合你：想快速浏览社区有哪些可直接复用的插件。

使用方式：

  • 当“插件商店目录”，按类别挑选。

3. MCP（Model Context Protocol）生态（推荐系统学习）

MCP 解决的是：模型如何以统一方式“接入工具/数据源/应用能力”。

3.1 modelcontextprotocol/modelcontextprotocol（规范）

  • 仓库：https://github.com/modelcontextprotocol/modelcontextprotocol

  • 用途：MCP 协议规范与文档。

  • 适合你：要理解 MCP 的抽象层（resources/tools/prompts、transport、capabilities）。

学习路径：

  • 先看 specification 的最小概念；

  • 再看官方 servers 里你感兴趣的数据源（比如 DB / 文件 / 网页转换等）。

3.2 modelcontextprotocol/servers（官方 Servers 合集）

  • 仓库：https://github.com/modelcontextprotocol/servers

  • 用途：官方 MCP servers 集合，覆盖常见能力。

  • 适合你：需要“拿来就用”的 server，或参考实现写自己的 server。

3.3 官方 SDK

  • TypeScript：https://github.com/modelcontextprotocol/typescript-sdk

  • Python：https://github.com/modelcontextprotocol/python-sdk

用途：

  • 快速开发 MCP server/client；

  • 与 Claude Desktop/Claude Code 等支持 MCP 的客户端打通。

4. 可视化 / UI（把 Claude Code 的过程“看得见”）

4.1 paulrobello/claude-office（像素办公室可视化）

  • 仓库：https://github.com/paulrobello/claude-office

  • 用途：实时像素办公室，把 Claude Code 的主 agent 与 subagent 行为可视化。

  • 建议：见上一章《Claude Office Visualizer 安装与使用》。

4.2 Yuyz0112/claude-code-reverse（交互可视化/分析）

  • 仓库：https://github.com/Yuyz0112/claude-code-reverse

  • 用途：把 Claude Code 与 LLM 的交互过程可视化/分析。

  • 适合你：想做“请求级别”的定位（例如某次输出异常、某次成本异常）。

4.3 seifghazi/claude-code-proxy（代理抓包 + 可视化）

  • 仓库：https://github.com/seifghazi/claude-code-proxy

  • 用途：作为代理捕获并可视化 Claude Code 的请求与对话。

  • 适合你：需要可观测性、想做审计或复盘。

4.4 preset-io/agor（多会话协作画布）

  • 仓库：https://github.com/preset-io/agor

  • 用途：在画布上编排 Claude Code/Codex/Gemini 的会话与任务，适合多人协作与并行。

5. 网关与观测（工程化落地常用）

5.1 BerriAI/litellm（多模型网关/代理）

  • 仓库：https://github.com/BerriAI/litellm

  • 用途：统一调用 100+ LLM（含 Anthropic），“OpenAI-compatible” 的代理层；可做路由、降级、成本/配额控制。

  • 适合你：

    • 想把 Claude 接入现有 OpenAI SDK/生态；
    • 需要多供应商容灾（Claude/GPT/Gemini 互备）；
    • 想做成本可视化与限流。

典型落地：

  • 在公司内网部署一个 LLM Gateway；

  • 上游应用统一走网关，不直接打供应商 API。

5.2 AgentOps-AI/agentops（Agent 观测与成本）

  • 仓库：https://github.com/AgentOps-AI/agentops

  • 用途：对 agent 运行做 tracing/监控/成本统计（多模型可接）。

  • 适合你：Agent 流程复杂，需要“像 APM 一样”观察每一步调用。

6. 代表性 MCP Servers（可直接补齐能力）

6.1 zcaceres/markdownify-mcp（内容转 Markdown）

  • 仓库：https://github.com/zcaceres/markdownify-mcp

  • 用途：把网页/HTML/各种内容转换为 Markdown，方便进入知识库或做 RAG。

6.2 bytebase/dbhub（数据库 MCP Server）

  • 仓库：https://github.com/bytebase/dbhub

  • 用途：对 Postgres/MySQL/SQL Server/MariaDB/SQLite 等提供统一的 MCP 数据库访问。

  • 适合你：希望 Claude 以“工具”的方式查库、写 SQL、生成报表。

6.3 QuantGeekDev/mcp-framework（写 MCP Server 的 TS 框架）

  • 仓库：https://github.com/QuantGeekDev/mcp-framework

  • 用途：更快搭一个 MCP server 的工程框架/脚手架。

7. 怎么选（给一个实用决策树）

  • 你只想“直接调用 Claude API”：先用官方 SDK（Python/TS）+ cookbook 抄模板。

  • 你在用 Claude Code 做工程：

    • 想要“可视化”：claude-office / claude-code-proxy。
    • 想要“会话沉淀/压缩”：claude-mem。
    • 想要“成本与用量管理”：ccusage。

  • 你需要把工具/数据源接入模型：系统学习 MCP，并选几款 servers 直接上。

  • 你要企业化落地：网关（LiteLLM）+ 观测（AgentOps）是一条常见路线。
